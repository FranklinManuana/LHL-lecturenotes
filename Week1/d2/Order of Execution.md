

Now that you have an idea of the parts of the SQL query, it is time to talk about how they all fit together. Each query starts with finding the data that you need from a database and then filtering that data down into something that can be understood quickly. Each part of the query is executed sequentially, hence, it's important to understand the order of execution so that you know what results are accessible and where.

Instruction

Read this article [about SQL Order of Execution](https://builtin.com/data-science/sql-order-of-execution) to familiarize yourself with the order of execution of an SQL query and to understand why it matters.

SQL queries adhere to a specific order when evaluating clauses. The query execution order optimizes your search results from the database and knowing the order of execution helps you better understand the errors.
